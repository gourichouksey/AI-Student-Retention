
# Security Policy

Thanks for helping keep **AI-Student-Retention** secure.

## Supported Versions

This project is community-maintained and may not publish versioned security releases.
Security fixes, when available, are applied to the default branch.

| Version / Branch | Supported |
| --- | --- |
| Default branch (latest) | Yes |
| Older commits / forks | No |

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security-sensitive reports.

### Preferred: GitHub private reporting / Security Advisory

If this repository has GitHub private vulnerability reporting enabled:

1. Go to the repository **Security** tab.
2. Click **Report a vulnerability**.
3. Provide a clear description and reproduction steps.

This is the fastest and most private way to report.

### If private reporting is not available

If you do not see **Report a vulnerability**:

1. Open a new issue **without including exploit details**, sensitive data, or proof-of-concept code.
2. In the issue, ask maintainers for a private contact method for security reports.
3. Once maintainers respond, share details privately.

### What to include

- Affected component(s) and impact
- Steps to reproduce (or a minimal proof-of-concept, shared privately)
- Any relevant logs, screenshots, or error messages (redact secrets)
- Suggested fix, if you have one

### Response targets (best effort)

- Acknowledgement within **7 days**
- Status update within **14 days**

Timelines can vary depending on maintainer availability and complexity.

## Coordinated Disclosure

We ask that you:

- Give maintainers a reasonable time to investigate and patch before public disclosure.
- Avoid accessing or modifying real user data during testing.

## Non-security Bugs

For non-security bugs, please open a regular GitHub issue.
